 
# Что такое документация в контексте программной разработки?
a) Совокупность текстовых и графических материалов, описывающих процесс и результаты разработки
b) Совокупность программных кодов и комментариев к ним
c) Совокупность отзывов и рекомендаций по использованию программы

**Ответ: a**

# Какие цели преследует документация при программной разработке?
a) Повышение качества и эффективности разработки
b) Повышение стоимости и сложности разработки
c) Повышение конкурентоспособности и популярности разработки

**Ответ: a**

# Какие виды документации можно выделить в процессе разработки?
a) Документация для заказчика, для разработчика, для пользователя
b) Документация для аналитика, для программиста, для тестировщика
c) Документация для менеджера, для дизайнера, для маркетолога

**Ответ: a**

# Какие аспекты необходимо учитывать при составлении документации?
a) Цель, аудитория, структура, стиль, формат
b) Цвет, шрифт, размер, отступы, выравнивание
c) Авторство, дата, версия, источники, ссылки

**Ответ: a**

# Какие инструменты используются для создания документации?
a) Текстовые редакторы, графические редакторы, системы управления версиями
b) Таблицы, диаграммы, схемы, графики
c) Бумага, ручка, карандаш, ластик

**Ответ: a**

# Какие правила следует соблюдать при написании технической документации?
a) Быть точным, ясным, лаконичным и последовательным
b) Быть творческим, оригинальным, эмоциональным и выразительным
c) Быть вежливым, уважительным, доброжелательным и внимательным

**Ответ: a**
